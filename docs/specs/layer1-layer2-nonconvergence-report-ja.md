## 7. なぜ「世に日本語パーサがある」のに未解決なのか
- 矛盾ではない。今回の課題は「一般的な最尤解析」ではなく、`Perl手順と同じ遷移系` に対する `到達可能性判定` である。
- 多くの実用パーサは、精度と時間のためにビーム・枝刈り・上限制御を使う。  
  これは「常に到達を見落とさない」保証と同値ではない。
- よって「パーサが存在する」事実だけでは、現在の到達失敗を反証できない。

## 8. 現在失敗の論理分解（4仮説）
Perl側で7手到達が既知であるため、Python側が未到達なら以下のいずれかが成立する。

1. 遷移関係不一致  
   - 候補列挙または規則適用が Perl と一致していない。
2. ゴール判定不一致  
   - `grammatical` 判定（未解釈素性カウント等）が Perl と一致していない。
3. 探索不完全  
   - DPOR/重複排除/visited/予算打切りにより、探索が完全でなくなっている。
4. 深さ・遷移集合定義不一致  
   - 「7手」の定義と Python 探索対象（単項操作の扱い等）が一致していない。

## 9. 最短の切り分け手順
1. `7手リプレイ`  
   - Perl手順書の各手を Python で逐次実行し、毎手で  
     `候補に存在するか / 適用後状態 / 未解釈数` を照合する。
2. `最小完全探索`  
   - DPOR・transposition を無効化した深さ7までの単純DLS/IDDFSを実行し、解の再発見可否を確認する。
3. 判定
   - 1で不一致が出れば「遷移系/判定不一致」問題。
   - 1が一致し2で発見できるなら「探索削減（DPOR/TT/予算）の副作用」問題。
   - 1が一致し2でも発見不能なら「深さ・遷移集合定義」問題。
