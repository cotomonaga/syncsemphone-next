# RH/LHマージ＋素性照合 と HPSG の共通点・相違点（正確化レポート）

更新日: 2026-02-27  
目的: RH/LHマージ＋素性照合とHPSGの関係を、誤解が出ない形で整理する。

## 1. 先に結論
1. 両者はどちらも「要素を結合して木構造を作る」点で共通する。  
2. ただし中核計算は異なる。  
   - RH/LH系: 規則適用による状態遷移（どの2要素をどう結合するかを操作で決める）  
   - HPSG: 型付き素性構造の統合（unification）とスキーマ制約で母を作る  
3. RH/LH系で非局所依存（長距離依存）を扱うことは可能であり、禁止ではない。  
4. 本プロジェクト文脈では、`grammatical` 到達は「未解消の解釈不可能素性が0になること」であり、これは抽象的には `need` の解消に対応する。

## 2. 共通の抽象骨格
両者とも抽象化すると次の形を持つ。

`if P(A, B) then C = F(A, B)`

- `P`: 結合可能性の判定
- `F`: 結合後の母構造を作る操作

違いは `P` と `F` の中身で生じる。

## 3. 何が違うか（中核計算）
### 3.1 RH/LHマージ＋素性照合
1. 操作の中心は `RH-Merge` / `LH-Merge`（左右指定つきの規則適用）。  
2. 適用後に素性を更新し、未解消の解釈不可能素性を減らしていく。  
3. 実装上は「状態遷移の列」として導出を追う。  
4. `grammatical` 判定は、未解消素性の残数が0かどうかで行う（本システムの運用）。

### 3.2 HPSG（典型的実装）
1. 操作の中心は、スキーマ（Head-Complement等）適用と型付き素性構造の統合。  
2. 統合で矛盾があれば失敗し、整合すれば情報を一つの構造に融合する。  
3. 実装上は「制約充足と統合」によって母構造を構成する。  
4. 局所・非局所情報を同一の素性体系で扱う（`NONLOC|SLASH` など）。

## 4. 「局所しかできない」は誤りか
はい、誤り。  
RH/LH系を局所照合に限定する必然はない。

1. RH/LH系でも、`gap/need` 相当の素性を設計して伝搬・解消すれば、長距離依存を表現できる。  
2. HPSGの `SLASH` 伝搬は、そのための代表的で体系化された実装様式。  
3. RH/LH側に同等の情報流通を実装することは可能で、理論的に禁止されない。  
4. 差は「できる/できない」より、どの程度標準化され、どれだけ宣言的に記述されるかにある。

## 5. need と gap の関係（本件の正確化）
本件での理解を正確に書くと次の通り。

1. 解釈不可能素性の残りは、抽象的には「未充足要求（need）」として読める。  
2. `grammatical` 到達は、その未充足要求が全て解消された状態に対応する。  
3. `gap` は通常、need のうち「非局所に持ち越される要求」を指す。  
4. したがって概念関係は一般に `gap ⊂ need`。  
   - すべての未解消素性が gap とは限らない。  
   - ただし本システム設計では、どの未解消素性を gap 扱いするかを明示的に決められる。

## 6. 具体例での比較
### 6.1 長距離依存
文: `この本を、太郎は花子が __ 読んだと言った。`

1. RH/LH系  
   - 空所対応を扱う専用素性（gap/need）を導入すれば導出可能。  
   - どの規則で伝搬し、どの規則で解消するかを実装側で定義する。
2. HPSG  
   - 典型的には `SLASH` を節境界で伝搬し、filler-head で解消する。
3. 差分  
   - 到達可能性そのものより、記述の標準化・再利用性・宣言性の差が大きい。

### 6.2 付加の係り先曖昧性
文: `太郎は公園で歌っている少女を見た。`

1. 読みA: 「公園で」が `歌っている` に係る（歌う場所=公園）  
2. 読みB: 「公園で」が `見た` に係る（見る場所=公園）

1. RH/LH系  
   - 両方の結合経路を構成し得る。  
   - どちらを許容/排除するかは素性制約の設計次第。
2. HPSG  
   - 付加スキーマと意味素性の統合で許容範囲を制約できる。
3. 差分  
   - どちらも曖昧性を表現可能だが、制約記述の形式が異なる。

## 7. 本プロジェクトへの実装含意
1. RH/LH方式を維持したまま、非局所依存を強化することは可能。  
2. その際は「未解消素性の型分け（局所need/非局所gap）」を先に定義する。  
3. 各規則で「どのneedを解消し、どのgapを上位へ伝搬するか」を明文化する。  
4. テストは次を固定する。  
   - 到達可能文で `grammatical` 到達すること  
   - 不適切導出で未解消素性が残ること  
   - 同一入力で履歴列が再現可能であること

## 8. まとめ
1. RH/LH＋素性照合とHPSGは、木を作るという目的は共通。  
2. しかし、計算の主役（規則適用中心か統合中心か）が異なる。  
3. RH/LHで `SLASH` 相当の技術を実装することは可能で、禁止されない。  
4. 本件では、`grammatical = 未解消の解釈不可能素性が0` を基準に、need/gapを明示設計することが最も実務的である。
