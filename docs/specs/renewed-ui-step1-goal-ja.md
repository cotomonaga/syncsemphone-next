# Renewed UI Step1ゴール定義

更新日: 2026-02-26

## Step1 のゴール

- 本ステップのゴールは「**観察に使う `.num`（Numeration）を1本確定すること**」です。
- `.num` は次の2系統の入口で扱います。
  - **Numerationを新規形成**（観察文から作る）
  - **既存 `.num` の読み込み**（保存済み/外部から持ち込んだものを読み込む）

## 導線（混在禁止）

- 観察文を使って新規に語彙を解決し `.num` を形成する入口は以下です。
  - `例文から選ぶ`（5例を選択）
  - `Lexiconから組み立てる`（自由入力観察文＋形態素解析）
- 既存 `.num` を持ち込む・読込する入口は以下です。
  - `numファイルを選ぶ`（ファイル選択＋テキスト貼り付け）
- どちらの入口でも、最終的に確認する主アクションは「`.num を作業中Numerationへ反映`」です。
  - 新規形成経路では、語彙解決→Numeration生成後に反映する。
  - 読込経路では、`.num` をそのまま反映する。

## 画面上の表示基準

- このステップの案内文において、`Numerationを作成` と `既存 .num を読込` が混在しないよう、ボタン文言と補助文言を分離して表示する。
- `.num` 形成系の手順は `生成前` と `読込前` を混同しない。
- 例文/アップロードが見えやすく、構文規則編集系の操作（Step2以降）と役割を混ぜない。

## 受け入れ基準（最小）

- 2経路の入口が視覚的に分かり、誤操作で別経路を実行しないこと。
- 既存 `.num` は「作成」ではなく「読込」の扱いとして扱われること。
- テキストベース観察文は `.num` 形成に必要な時だけ有効化されること。
- Step1 のメインボタンと補助説明が、現在の経路と一致していること。
